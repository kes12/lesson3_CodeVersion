Highlight #1 Developer Tools

Developer tools shows the http response to an http request; it shows what the page sends to the browser. For example, the common file not found error 404 message appears if the numbers being sent and received are incorrect. Developer tools also make available all of the code for the whole page allowing you to see the css style. You may also edit and change to webpage for trial and error to see what would look best. Lastly, developer tools also shows the sources used to generate the page; css and html.

Highlight #2 Firewall and Proxy

Firewalls and Proxys are very similar in that they both search for misplaced packets and decide whether or not a package is acceptable or not. A proxy can be viewed as a specialized firewall; it is used more specifically for certain apps. A proxy understands the protocol it is designed for, such as HTTP. A firewall cannot accept and deny things baseed on protocols, instead it accepts or denies based on thins such as a mac or IP address.

Highlight #3 The Process of the Internet
